# 修饰符内容漏洞

```solidity
  modifier onlyOwner() {

        require(mask & bytes20(msg.sender) == mask & bytes20(owner));

        _;

    }   
```

修饰符里是在对地址做按位**与(AND)运算**后再比较,也就是说**只要 **`msg.sender`** 在按位 AND 后的结果跟 **`owner`** 一样就可以通过检查**,这很坏了

* <font style="background-color:#f3bb2f;">与运算</font>：两个变量中**都是1**的位，其`与运算`结果位**才是1**，否则都是0
* 也就是看的位置写1,把不看的位置都变成0

就意味着只要mask被检查的位置都和msg.sender一致,就能成为owner,其他的位置蒙混过关了

最安全常见的做法：

```solidity
modifier onlyOwner() {
    require(msg.sender == owner, "Not owner");
    _;
}
```
