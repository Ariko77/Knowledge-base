# 槽

在 Solidity 中，每个合约都有一个链上的存储（storage），可以类比为一个大数组，每个格子叫一个**槽**（slot），大小为 **32 字节（256 位）**,状态变量就被按顺序塞进这些槽里

### 规则

| 情况 | 存储方式说明 |
| --- | --- |
| 简单变量（如 `uint256`, `address`） | 顺序占用 slot，1 个变量 1 个槽（除非可以压缩） |
| 小变量（如 `uint8`, `bool`） | 尽可能压缩进一个槽里，从高位向低位排 |
| `mapping` / `dynamic array` / `string/bytes` | 不存储值在 slot 中，只存一个“入口槽”，具体值用 \*\*`keccak256` \*\*算位置 |

* uint256: 32 字节
* uint8: 1 个字节
* bool: 1 个字节
* address: 20 个字节
* string: 占据空间长度取决于字符串，每个合约的状态变量都是按照槽位（slot）的顺序排列存储的

### 读取槽

```solidity
cast storage 实例地址 几槽 --rpc-url $...
```
