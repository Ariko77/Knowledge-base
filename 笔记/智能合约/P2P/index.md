# P2P

<font style="color:rgb(18, 18, 18);"> P2P（Peer to Peer）= 节点之间直接通信，没有中心服务器。以太坊是一个点对点网络，拥有数千个节点，这些节点必须能够使用标准化协议相互通信。“网络层”是一系列协议的集合，它允许这些节点相互发现并交换信息。这包括在网络上“传播”信息（一对多通信），以及在特定节点之间交换请求和响应（一对一通信）。每个节点都必须遵守特定的网络规则，以确保它们发送和接收的信息是正确的。 </font>

## 区块链为<font style="color:rgb(18, 18, 18);">执行层的网络协议分为两个协议栈：</font>

* <font style="color:rgb(18, 18, 18);">发现协议栈：构建于 UDP 之上，允许新节点找到要连接的对等节点。</font>
* <font style="color:rgb(18, 18, 18);">DevP2P协议栈：位于TCP之上，使节点能够交换信息。</font>

<font style="color:rgb(18, 18, 18);">这两个协议栈并行运行。发现协议栈负责将新的网络参与者引入网络，而 DevP2P 协议栈则支持他们之间的交互。</font>

## 什么离不开 P2P？

1. **去中心化传播区块和交易**
   * 当有人挖出新区块时，它要通过 P2P 网络广播给其他节点。
   * 不存在一个“主服务器”负责通知大家。
   * 所以 P2P = 区块链的“消息传输层”。
2. **每个节点都是平等的**
   * 不管是你自己运行的节点、矿工、还是验证者，理论上地位一样。
   * 都能自己决定信任谁、连接谁。
3. **抗审查能力强**
   * 因为消息不是经过一个中心，所以想“封掉”整个区块链几乎不可能。
4. **健壮性高**
   * 某个节点挂了，其他节点还能互联互通。

## 区块链里的 P2P 是怎么组织的

可以想象为一个“**松散的社交网络**”，节点靠一些机制找到彼此：

| 功能 | 机制 | 举例 |
| --- | --- | --- |
| **节点发现** | 互相介绍、记录种子节点、DNS 启动 | Bitcoin 的 `addr`<br/> 协议、以太坊的 Discovery v5 |
| **消息传递** | Gossip（八卦传播） | 比特币、以太坊都是这种：每个节点只告诉邻居，再层层扩散 |
| **数据存储** | DHT（分布式哈希表） | IPFS、Filecoin 用这个 |
| **连接方式** | TCP / UDP / WebSocket / QUIC | 不同链各有优化 |

## 区块链项目中常见的 P2P 应用

| 项目 | P2P 用途 |
| --- | --- |
| **Bitcoin** | 广播交易和区块、节点发现 |
| **Ethereum** | 同上，但协议更复杂（RLPx、Discovery v5） |
| **IPFS / Filecoin** | 存储文件，用 libp2p 管理节点和数据路由 |
| **Polkadot / Cosmos** | 用 Gossip 和 libp2p 做跨链通信 |
| **BitTorrent** | 文件共享的鼻祖，后来 Filecoin 借鉴了它的结构 |

## P2P 和区块链之间的“关系图”

```plain
┌────────────────────────┐
│   用户 & 钱包 (RPC)    │
└────────────┬───────────┘
             │
┌────────────▼────────────┐
│ 区块链节点（执行合约） │
│ - 同步区块              │
│ - 校验交易              │
│ - 存储状态              │
└────────────┬────────────┘
             │ P2P 网络层
┌────────────▼────────────┐
│     节点互连、消息广播   │
│     (libp2p / Gossip)   │
└─────────────────────────┘
```

区块链其实是“**P2P 网络 + 共识协议 + 状态机**”。

## Q\&A

1. **节点如何互相连上**（靠种子节点 + P2P 协议）
2. **消息如何传播**（靠 Gossip，像八卦）
3. **为什么它能去中心化**（因为没有单点）
4. \*\* IPFS / libp2p 是谁在用的组件\*\*（以太坊、Filecoin）
