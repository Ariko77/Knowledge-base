# 变量存储 紧密打包

## 数据存储

1. `storage`：合约里的**状态变量**默认都是`storage`，存储在**链上**。***消耗gas多***
2. `memory`：函数里的**参数**和**临时变量**一般用`memory`，存储在**内存中，不上链**。<font style="background-color:#f3bb2f;">尤其是如果返回数据类型是变长的情况下，必须加memory修饰</font>，例如：**string, bytes, array**和**自定义结构**。***消耗gas少***
3. `calldata`：和`memory`类似，存储**在内存中，不上链**。与`memory`的不同点在于`calldata`变量**不能修改**（`immutable`），一般用于函数的参数。***消耗gas少***

* uint256: 32 字节
* uint8: 1 个字节
* bool: 1 个字节
* address: 20 个字节
* string: 占据空间长度取决于字符串，每个合约的状态变量都是按照槽位（slot）的顺序排列存储的

## 变量紧密打包

当多个较小的变量（例如：uint8, uint16, bool 等）连续声明时，编译器会将多个变量紧密打包到同一个槽位中，以节约空间

* Solidity 会尽量把连续声明的小变量塞进同一个32字节槽位中
* 如果超出32字节，则另开新槽位

```solidity
contract PackedStorage { 
    uint128 a; // slot 0 (前16字节) 
    uint64 b; // slot 0 (中间8字节) 
    uint64 c; // slot 0 (最后8字节) 
    uint256 d; // slot 1 (新槽位) 
}
```

例如

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract StorageSlotExample {
    uint256 a = 1;      // slot 0
    uint128 b = 2;      // slot 1（与 c 共用）
    uint128 c = 3;      // slot 1
    uint8 d = 4;        // slot 2（与 e 共用）
    bool e = true;      // slot 2
}
```

```solidity
bytes1 <==> uint8 //1字节 <==> 8位
bytes32 <==> uint256 //32字节 <==> 256位
address <==> bytes20 //address <==> 20字节
address <==> uint160 //address <==> 160位

//address转换为uint256
address a = 0x;
uint b;

```
