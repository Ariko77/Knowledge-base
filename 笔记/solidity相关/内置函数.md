# 常用内置函数

## 合约上下文相关（msg/tx/block）

| 语法 | 说明 |
| --- | --- |
| **<font style="background-color:#f3bb2f;">msg.sender</font>** | 当前调用合约的地址（攻击者） |
| `msg.value` | 发送的 ETH 数量（单位：wei） |
| `msg.data` | 调用数据 payload（字节数组） |
| **<font style="background-color:#f3bb2f;">tx.origin</font>** | <font style="background-color:#f3bb2f;">最初</font>发起交易者地址 |
| `block.timestamp` | 当前区块时间戳（秒） |
| `block.number` | 当前区块高度 |
| `blockhash(uint)` | 给定区块号的区块哈希 |

## 地址与转账操作(address)

| 语法 | 说明 |
| --- | --- |
| `address(this)` | 当前合约地址 |
| `address(this).balance` | 当前合约余额（单位：wei） |
| `payable(address).transfer(x)` | 转账，失败会自动 revert（2300 gas） |
| `payable(address).send(x)` | **返回布尔值**，不会自动revert |
| `address.call{value: x}("")` | **低级调用**，推荐搭配 `require(success)` |

## 哈希与加密工具(仅整理,这里还没学到过)

| 语法 | 说明 |
| --- | --- |
| `keccak256(bytes memory)` | Solidity 的 sha3 |
| `sha256(bytes memory)` | SHA-256 哈希 |
| `ripemd160(bytes memory)` | RipeMD-160 哈希 |
| `blockhash(uint blockNumber)` | 获取历史区块哈希 |

## ABI 编解码

| 语法 | 说明 |
| --- | --- |
| `abi.encode(...)` | ABI 编码成字节流 |
| `abi.encodePacked(...)` | 紧凑编码，适合 keccak256 使用 |
| `abi.decode(bytes, (Type1, Type2))` | 解码 ABI 数据 |

## 类型转换与构造函数

| 语法 | 说明 |
| --- | --- |
| `uint8(x)` | 强制转换为 uint8 |
| `bytes32(uint256(x))` | 转换为固定大小字节数组 |
| `type(MyContract).creationCode` | 获取合约部署代码（bytecode） |
| `new MyContract(...)` | 新部署一个合约 |

## 合约销毁

| 语法 | 说明 |
| --- | --- |
| `selfdestruct(payable(address))` | 销毁合约并发送 ETH 到地址 |

## 函数选择器工具（常见于 calldata 操作）

| 语法 | 说明 |
| --- | --- |
| `bytes4(keccak256("funcName(args)"))` | 函数选择器，用于**低级调用** |

## EVM 低级原语（常见于汇编操作和底层判断）

| 语法 | 说明 |
| --- | --- |
| `extcodesize(address)` | 获取某地址的代码大小（合约部署前为 0） |
| `extcodehash(address)` | 获取某地址的代码哈希（判断是否合约） |
| `isContract(address)` | 通常用 extcodesize 判断合约与否 |
| `selfbalance()` | 当前合约余额（与 this.balance 类似） |
| `codecopy(...)` | 复制代码段（Yul 或内联汇编使用） |
| `create(...)` | 创建合约（更底层的 `new`） |
| `create2(...)` | 用特定地址部署（可预测地址） |

## Solidity 特殊关键字与函数（易忽视但常用）

| 语法 | 说明 |
| --- | --- |
| `constructor()` | 构造函数，部署合约时运行一次 |
| `receive()` | 接收 ETH（无 calldata 时调用） |
| `fallback()` | 默认函数（无函数匹配或 calldata 存在） |
| `revert(reason)` | 回滚并返回错误信息 |
| `require(condition, reason)` | 条件不满足就回滚 |
| `assert(condition)` | 内部一致性检查失败时回滚 |
| `unchecked {}` | 不进行溢出检查（gas 更省） |

## 数据编码与类型操作（高级数据处理）

| 语法 | 说明 |
| --- | --- |
| `abi.encode(...)` | 编码成 ABI 格式 |
| `abi.encodePacked(...)` | 编码为紧凑格式，适合哈希处理 |
| `abi.decode(bytes, (T1, T2))` | 解码 bytes 数据 |
| `bytes4(keccak256("f(uint256)"))` | 函数选择器 |
| `type(MyContract).creationCode` | 获取合约创建字节码 |
| `type(MyContract).runtimeCode` | 获取合约运行字节码（一般调试用） |
| `uint160(address)` | 把地址转成 uint |
| `uint32(uint64(x))` | 多级类型强转，常用于精确操作位段 |

## 合约调用和转账相关低级函数（常用于攻击与绕过）

| 语法 | 说明 |
| --- | --- |
| `address.delegatecall(data)` | 在当前合约上下文运行目标合约代码 |
| `address.staticcall(data)` | 只读调用 |
| `selfdestruct(address)` | 销毁当前合约并转移所有 ETH |

## tips

* `abi.encodePacked` 与 `keccak256` 搭配用，构造特定哈希。
* `call` 返回 `(bool success, bytes memory result)`，记得检查 `success`。
* `type(MyContract).creationCode` 常用于创建合约时的 bytecode 获取。
* `selfdestruct` 会把合约内所有 ETH 发出去并删除代码
