# 权限管理漏洞

<font style="color:rgb(44, 62, 80);">智能合约中的权限管理定义了不同角色在应用中的权限。通常来说，代币的铸造、提取资金、暂停等功能都需要较高权限的用户才能调用。如果权限配置错误，就可能造成意想不到的损失。下面我们介绍两种常见的权限管理漏洞:</font>

# <font style="color:rgb(44, 62, 80);">权限配置错误</font>

<font style="color:rgb(44, 62, 80);">如果合约中特殊功能没有加上权限管理，那么任何人都能铸造大量代币或将合约中的资金提光。</font>

<font style="color:rgb(44, 62, 80);">在下面的代码中，</font><code><font style="color:rgb(71, 101, 130);background-color:rgba(27, 31, 35, 0.05);">mint()</font></code><font style="color:rgb(44, 62, 80);">函数没有进行权限管理，那么任何人都可以调用它铸造代币。</font>

```solidity
//错误的mint函数，没有限制权限
function badMint(address to, uint amount) public {
    _mint(to, amount);
}
```

# 授权检查错误

<font style="color:rgb(44, 62, 80);">另一类常见的权限管理漏洞是没有在函数中检查调用者是否拥有足够的授权。BSC上DeFi项目 ShadowFi 的代币合约忘了在 </font><code><font style="color:rgb(71, 101, 130);background-color:rgba(27, 31, 35, 0.05);">burn()</font></code><font style="color:rgb(44, 62, 80);"> 销毁函数中</font><font style="color:rgb(44, 62, 80);background-color:#FBDFEF;">检查调用者的授权额度</font><font style="color:rgb(44, 62, 80);">，导致攻击者可以任意的销毁其他地址的代币。在黑客将流动性池子中的代币销毁之后，仅需卖出一点代币就可以将池子里的所有 </font><code><font style="color:rgb(71, 101, 130);background-color:rgba(27, 31, 35, 0.05);">BNB</font></code><font style="color:rgb(44, 62, 80);"> 提走，获利 $300,000。</font>

```solidity
// 错误的burn函数，没有限制权限
function badBurn(address account, uint amount) public {
    _burn(account, amount);
}
```

# <font style="color:rgb(44, 62, 80);">预防办法</font>

<font style="color:rgb(44, 62, 80);">权限管理漏洞主要有两种预防办法：</font>

1. <font style="color:rgb(44, 62, 80);">使用 Openzeppelin 的权限管理库给合约的特殊函数配置相应的权限，比如使用</font><code><font style="color:rgb(71, 101, 130);background-color:rgba(27, 31, 35, 0.05);">OnlyOwner</font></code><font style="color:rgb(44, 62, 80);">修饰器，只有合约所有者才能调用。</font>

```solidity
// 正确的mint函数，使用 onlyOwner 修饰器限制权限
function goodMint(address to, uint amount) public onlyOwner {
    _mint(to, amount);
}
```

1. <font style="color:rgb(44, 62, 80);">在函数的逻辑中确保合约调用者拥有足够的授权。</font>

```solidity
// 正确的burn函数，如果销毁的不是自己的代币，则会检查授权
function goodBurn(address account, uint amount) public {
    if(msg.sender != account){
        _spendAllowance(account, msg.sender, amount);
    }
    _burn(account, amount);
}
```
