# 4. accrueInterest借款的更新 利率的计算

```solidity
 function accrueInterest() public virtual override returns (uint256) {
        /* Remember the initial block number */
   // 查看区块的利率
        uint256 currentBlockNumber = getBlockNumber();
   
        uint256 accrualBlockNumberPrior = accrualBlockNumber;

        /* Short-circuit accumulating 0 interest */
        if (accrualBlockNumberPrior == currentBlockNumber) {
            return NO_ERROR;
        }

        /* Read the previous values out of storage */
        // cashPrior：市场中 token 的余额
        // borrowsPrior：借贷总额（欠钱总额）
        // reservesPrior：储备金
        // borrowIndexPrior：上次记录的借款指数
        uint256 cashPrior = getCashPrior();
        uint256 borrowsPrior = totalBorrows;
        uint256 reservesPrior = totalReserves;
        uint256 borrowIndexPrior = borrowIndex;

        /* Calculate the current borrow interest rate */
        uint256 borrowRateMantissa = interestRateModel.getBorrowRate(cashPrior, borrowsPrior, reservesPrior);
        require(borrowRateMantissa <= borrowRateMaxMantissa, "borrow rate is absurdly high");

        /* Calculate the number of blocks elapsed since the last accrual */
        uint256 blockDelta = currentBlockNumber - accrualBlockNumberPrior;

        /*
         * Calculate the interest accumulated into borrows and reserves and the new index:
         *  simpleInterestFactor = borrowRate * blockDelta
         *  interestAccumulated = simpleInterestFactor * totalBorrows
         *  totalBorrowsNew = interestAccumulated + totalBorrows
         *  totalReservesNew = interestAccumulated * reserveFactor + totalReserves
         *  borrowIndexNew = simpleInterestFactor * borrowIndex + borrowIndex
         */

        Exp memory simpleInterestFactor = mul_(Exp({mantissa: borrowRateMantissa}), blockDelta);
        uint256 interestAccumulated = mul_ScalarTruncate(simpleInterestFactor, borrowsPrior);
        uint256 totalBorrowsNew = interestAccumulated + borrowsPrior;
        uint256 totalReservesNew =
            mul_ScalarTruncateAddUInt(Exp({mantissa: reserveFactorMantissa}), interestAccumulated, reservesPrior);
        uint256 borrowIndexNew = mul_ScalarTruncateAddUInt(simpleInterestFactor, borrowIndexPrior, borrowIndexPrior);

        /////////////////////////
        // EFFECTS & INTERACTIONS
        // (No safe failures beyond this point)

        /* We write the previously calculated values into storage */
        accrualBlockNumber = currentBlockNumber;
        borrowIndex = borrowIndexNew;
        totalBorrows = totalBorrowsNew;
        totalReserves = totalReservesNew;

        /* We emit an AccrueInterest event */
        emit AccrueInterest(cashPrior, interestAccumulated, borrowIndexNew, totalBorrowsNew);

        return NO_ERROR;
    }
```

## 区块的计算
在一开始先计算当前的区块是多少

```solidity
uint256 currentBlockNumber = getBlockNumber();
uint256 accrualBlockNumberPrior = accrualBlockNumber;
```

然后俩值相减就会得到距离上次结算经过了多少区块

## 简单利息因子 simpleInterestFactor
simpleInterestFactor  =  borrowRate（每个区块的利率）  * 经过区块数

例：

+ borrowRate = 1e12 = 0.000001
+ blockDelta = 1000

simpleInterestFactor = 1e12 * 1000 = 1e15   （表示 0.001）

意思是在1000 个区块里面 全市场的欠款增长率是0.1%

## 计算利息总额（interestAccumulated），并加入借款池
```solidity
interestAccumulated（ 这一段时间产生的总利息  ） = borrowsPrior （池子里的总借款数量）* simpleInterestFactor（协议定的）
```

例：

+ borrowsPrior = 1000
+ simpleInterestFactor = 0.001

interestAccumulated = 1000 * 0.001 = 1

全市场产生了1个原生代币的利息

## 更新借款总额
```solidity
uint256 totalBorrowsNew = interestAccumulated + borrowsPrior;
```

## 从利息中抽取储备金
新增储备金 = interestAccumulated * **reserveFactor（协议定的）**

新的 reserves = reservesPrior（原先的储备金） + 新增储备金

## 计算新的BorrowIndex
borrowIndexNew = borrowIndexPrior * (1 + simpleInterestFactor)

```solidity
用户需要还款数=借款*borrowIndexNew/borrowIndexPrior
```

上下会抵消，只剩下那个(1 + simpleInterestFactor)起作用，所以BorrowIndex一直在变大也没什么，不会让用户还的钱越来越多，而且在真正的池子里位数是很多的，变化微乎其微

 这一点代表未来所有用户欠款的增幅都由 borrowIndex 体现。  



详细解释BorrowIndex含义

现实例子：

银行会借贷很多人不可能为每个人时刻更新利息，但是可以保存一个数，让这数字的变化来记录例子的变化。

比如 我借了1000 块 现在的BorrowIndex假如现在的Borrowindex 是1 

然后我很长时间都不管，我并不知道我具体利息多少，但是Borrowindex是更新的

比如说某天我来还款，Borrowindex变成了1.01

所以我们需要还款的数量就是 1000 * 1.01 / 1 = 1010

这就是BorrowIndex在compound里面的应用

需要注意的是BorrowIndex不会变小，永远不会变小
